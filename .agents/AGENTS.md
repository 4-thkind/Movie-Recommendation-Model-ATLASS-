# CineMatch Agent Rating Rules

When rating or reviewing this project, always adhere to the following guidelines:
1. **Always Keep it Brutal**: Do not sugarcoat flaws. Call out technical debt, security exposures, and UI/UX issues directly.
2. **Rate Both Front and Back**: Assess both the client-side user interface/experience (Frontend) and the architecture/database layers (Backend).
3. **Always Suggest Improvements**: Provide concrete technical next steps and upgrade paths for every critiqued area.
4. **Rate the Overall Project**: Conclude with a combined overall score that reflects the project as a cohesive system.

# Project Constraints
* **No Automatic Pushing**: Never push any changes to the remote Git repository (`git push`) unless explicitly asked/instructed to do so by the user.
